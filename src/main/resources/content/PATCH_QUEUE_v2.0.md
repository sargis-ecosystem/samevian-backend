## 🤖 Մաս 6 · Արհեստական Բանականություն (AI)-ին Ուղղված Հրահանգ

### 6.1 · Հրամանների Ցանկը

| Հրաման | Իմաստ | Արձագանք |
|---|---|---|
| `sync` | Կարդա Queue-ն, ստեղծի Plan | Ցուցադրել Patch Plan |
| `sync #N` | Միայն N համարի Patch-ը | Ցուցադրել միայն N-ի Plan |
| `sync all` | Բոլոր Pending-ները | Ցուցադրել ամբողջ Plan |
| `patch list` | Ակտիվ Queue-ն ցուցադրել | Աղյուսակ ⏳ Pending-ից |
| `patch done #N` | N-ը նշել ավարտված | Տեղափոխել ✅-ի մեջ + CHANGELOG |
| `patch reject #N` | N-ը մերժել | Տեղափոխել ❌ + պատճառ |
| `patch status` | Ընդհանուր կարգավիճակ | Ամփոփ՝ X pending, Y done, Z rejected |

### 6.2 · Patch Plan-ի Ձևաչափը

Երբ Սարգիսը գրում է `sync`, Արհեստական Բանականություն (AI)-ը պատասխանում է հետևյալ կառուցվածքով.

```markdown
## 🔄 PATCH PLAN · [Ամսաթիվ]

### Patch #N · [Կարճ վերնագիր]

**Ազդեցության շրջանակը՝**
- 📄 [Ֆայլ 1] → §[Բաժին] → [Կոնկրետ փոփոխություն]
- 📄 [Ֆայլ 2] → §[Բաժին] → [Կոնկրետ փոփոխություն]

**Cross-references-ի թարմացում՝**
- SARGIS_ECOSYSTEM Մաս [X] → [Փոփոխություն]

**Version bump՝**
- [Ֆայլ] → [Հին տարբերակ] → [Նոր տարբերակ]

**CHANGELOG entry՝**
## [vX.X] — YYYY-MM-DD · [Patch անվանումը]
✅ [Փոփոխությունը · 1 տող]

**Կարծես ճի՞շտ է։ ✅ Հաստատել · 🟡 Մասնակի · ❌ Մերժել · ✏️ Ճշտել**
```

### 6.3 · Ինքնապաշտպանական Կանոններ

| Կանոն | Կիրառություն |
|---|---|
| **Ոչ ինքնագործ sync** | Մինչև Սարգիսը չգրի `sync`, ոչինչ չի փոխվում |
| **Ոչ silent edit** | Ամեն փոփոխություն ցուցադրվում է **նախքան** կիրառելը |
| **Confirmation պարտադիր** | Մինչև «✅ Հաստատել», փաստաթուղթը չի թարմացվում |
| **Ատոմիկ կիրառում** | Կամ ամբողջ Patch-ը կիրառվում է, կամ ոչ մեկը (no half-state) |
| **CHANGELOG ինքնահարկադիր** | Ամեն ✅ Done → CHANGELOG entry ավտոմատ |
| **Version bump կանոն** | Major change → minor version (v4.4 → v4.5) · Critical → major (v4.4 → v5.0) |
